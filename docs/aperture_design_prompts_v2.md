Copy-paste prompts for **Claude Design** (or any Claude that outputs UI as a single HTML file) to
design every screen of the converged Aperture app — look, layout, components, styles.

Version 2.0 · 2026-06-13 · Supersedes [aperture_design_prompts.md](aperture_design_prompts.md).
Design source of record: [aperture_v1_design.md](aperture_v1_design.md).

> **What's new in v2:** the full converged inventory (all 16 web screens incl. Signals, Portfolio,
> Execution & TCA, Monitoring, Track Record, Scenarios, Replay, Preflight, Audit, Settings), the
> iOS decision-focused subset, a States sheet with the honest "coming"/DataUnavailable treatments,
> and one locked design system + one sample dataset across every prompt (so the screens form a
> single coherent app). Every prompt was machine-checked to be self-contained, on-system, and
> sample-data-consistent.

---

## How to use this pack

1. **Start a Claude Design conversation and paste `0 · FOUNDATION` first.** It defines the brand,
   the full design system, the component vocabulary, the decision-chain concept, and the sample
   data. Keep it in context for the whole session (re-paste at the top if the chat gets long).
2. **Paste `1 · STYLE GUIDE` next** to lock the visual language as a single artboard.
3. **Then paste one screen prompt at a time.** Each is self-contained (it restates the specific
   sample data + components it needs) but assumes FOUNDATION was pasted first.
4. **For consistency**, design in order and tell Claude *"match the exact visual language, spacing,
   and colors of the previous screen."*
5. **Iterate in place** — after a mockup, ask for targeted changes (*"hero chart +60px; move the
   regime card above ideas"*) rather than regenerating.
6. **Variations** — append *"give me a denser second version"* or *"a light-theme variant."*

Screens render the **full realized vision** with rich mock data (that's the point of a mockup).
The honest empty / loading / stale / **coming** states live in the final **States** prompt — those
are the treatments the shipped app uses where data isn't wired yet (see
[data_readiness.md](data_readiness.md)).

## Tips for great results

- **Order matters.** FOUNDATION → STYLE GUIDE → web screens in listed order → iOS → States.
  Locking the style guide early makes every later screen match.
- **Force consistency.** Start later prompts with *"Match the exact visual language, spacing, and
  colors of the previous screens."*
- **One coherent app.** Every prompt uses the same NAV ($1.21M), the same 9 ideas, the same prices
  and strategies — so screenshots line up into one product.
- **Export.** Ask for *"a single self-contained HTML file"* (already requested), screenshot at the
  canvas size for your design library, or feed back into the prototype in `apps/prototype/`.
- **Ground further (optional).** Attaching [aperture_v1_design.md](aperture_v1_design.md) and a
  couple of prototype screenshots makes output even more faithful, but isn't required.

*"Aperture" is a placeholder brand; palette and naming are easy to change. This pack and the
prototype are meant to be iterated together.*
